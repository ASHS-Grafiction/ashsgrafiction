Creating Pages


Creating Pages

Open Bootstrap Studio. You'll see the File button at your top-left. Click it and a dropdown menu will fall. Click open and open your downloaded copy of Grafiction's source file. The source file should have the extension .bsdesign.

I'm not going to try and teach the entirety of Bootstrap Studio. I will however, drop this link to the official documentation and there's also the tutorial and examples built into Bootstrap Studio you can use (it can be seen when you open the app as the Tutorials tab next to Recent Designs).

Before moving on however, I'll emphasize the need to turn on Automatic Backups as accidents do and will happen. To do so, open the File menu again but hover on Backups and click Backup Settings. If Automatic Backups aren't on, turn them on and set the frequency to whatever you prefer.


Source Control

Source control refers to the system we use to track and keep ourselves up-to-date with all the changes made to the website. The process of making changes, exporting those changes, and uploading them depends on which process we're doing, creating pages or uploading comics.

With pages, once you have the page looking the way you want it and you've previewed it in action you're ready to upload it to our source control system: Github. Here projects are stored in what we call repositories. You're going to be making changes to our website's repository.

First, save the project (by hitting Ctrl+S or ⌘+S) then click on the arrow next to Export (to your top-left). A drop-down menu should show then click Export Settings. Make sure to set your settings to the following:

The exception to this being the Export Destination.

Set that to where your computer has the Github repository stored. You can find out where by opening Github Desktop (and while in the ASHS Grafiction repository) clicking on the button encircled below:

Now you can click Export and Bootstrap Studio will make the appropriate changes to the repository. After setting your settings once, you can skip this step from here on and just click Export as Bootstrap Studio will just use your last used settings.

The repository on your computer has been updated but there's still the task of uploading this to the online copy of the website. Go back to Github Desktop, there should be a list of changes made that looks something like this:

There should also be a form stating Summary (required). Just write a very short description (1 fragment or sentence) that describes what you changed (misc. improvements, design fixes, added new comic, etc.). Don't do this:

Because a descriptive summary will give you clues as to what has changed over time in case something goes wrong. If something does go wrong, we can revert to a previous version of the website. I'll put the instructions for that in the FAQs.

However, right now we're not reverting changes we're pushing them. Once you've written what you need, hit Commit. Then once it's done, press Push Origin. Wait for the files to be uploaded and the public website itself will update with your changes in at most 5 minutes. Congrats! You've changed the website. Make sure to go to the website to make sure nothing catastrophic has happened (this occasionally happens where a change will look fine when previewed but transform dramatically when uploaded).

Also make sure that if someone else has made changes and pushed them that you later Pull Origin before making any new changes (this is like SUPER IMPORTANT). Those changes you made were also applied to the .bsdesign file so make sure to upload that new version to your shared GDrive.

Now that we know how to edit and make new pages, let's actually put some comics on those pages.


Uploading Comics


Preparations 

First the comic has to be stored in a folder where the images of the comic are named in sequential order (01.png, 02.png, 03.png...). Make sure however (and make sure the comic's submitter is also aware) that the filenames of the images cannot start with the number. Either have leading zeroes prior to the number (001.png, 012.png, 130.png...) as many zeroes as can fit the largest number in the filenames or simply put descriptive text before the number (titleofcomic_1.png, titleofcomic_2.png...) just make sure the text stays the same. If I were telling members how to submit their comics for say the anthology, I'd just tell them to name it as:

page_1.png, page_2.png, page_3.png, and so on...

To avoid overcomplicating it. I wrote the alternatives above in case that's preferable for you but I'll also put the technical explanation for why naming files like:

1.png, 2.png, 3.png, and so on...

...can be catastrophic in the FAQs.

Source Control (continued)

The process for uploading changes from Bootstrap Studio and from Lue Comic Generator work the same way. Export, Commit, Push. Exporting however isn't quite done after hitting the Export button when uploading comics however.

After your generated comic is to your liking you can finalize the export by hitting Ctrl+S or ⌘+S while you have the exported comic opened in your web browser. A file dialog should pop up asking you where to save the file. Navigate to the ASHS Grafiction repository on your computer and then open up the comics folder. Rename the file and then click save. The format thus far for the renamed file is "lowercasetitleofcomicwithoutanyspaces.html" but that's open to change. Just choose a format and stick to it, maybe edit this paragraph once you do.

Then just Commit then Push. Finished.

Thumbnail and description

On the comics page of ASHS Grafiction (comics/index.html), all the comics and their thumbnails are arranged in a grid. In Bootstrap Studio, the grid is made up of elements called Cards.

To add a new thumbnail you need to make a new row as well so right click on one of the Card Groups in the Overview (top left) and right-click it and click Duplicate (you could also hit Ctrl+D or ⌘+D. Then edit one of the Cards. Change the description, author, and thumbnail until you're happy and now we just need to change what URL the thumbnail brings you to when you click it.

We're going to have to go into the code for this one but this'll be the only time. Ever. I'll walk you through it so just pay attention.

Drag up the HTML panel next to the Styles panel at the bottom of your screen. There'll also be an Attributes section below the HTML panel. There'll be a lot of code but don't worry just focus on the part where the blue box is. Does it say class="card" or class="card-body" anywhere? If so, you're on the right track. Otherwise, go back and select the card again.

Now, click on the line that says img class="card... and it should look something like this:

We're going to edit one of the attributes but only one so I've blurred out the rest because you can ignore all of that. Just focus on onclick.

That part next to it is the code the runs whenever someone clicks or taps on the thumbnail and it sends them to the correct URL. We're going to edit that code but just the URL to keep it simple. You can do this in a separate window maybe in a text editor (just not Google Docs or Microsoft Word because those tend to mess with code for reasons I'll put in the FAQs) or in the form box itself.

The code is:

if(!thisisaphone() && firstTap){ window.open('https://ashsgrafiction.com/comics/untitled'); } else if (thisisaphone() && firstTap){ firstTap = false; } else if (!firstTap){ window.open('https://ashsgrafiction.com/comics/untitled'); }

I've highlighted the part you're changing. You don't even need to change the whole URL out just change the part after /comics/ to the format you used earlier to export the comic you're linking to. Once that's been changed out you're done. Just remember that the URL appears twice so change it twice.

PSA: This is due to me having a small lapse in judgement while writing this code early on that I'm only caught now. If need be try replacing the code in there with this code. It hasn't been tested though so I don't recommend it.

url='https://ashsgrafiction.com/comics/untitled';if(!thisisaphone() && firstTap){ window.open(url); } else if (thisisaphone() && firstTap){ firstTap = false; } else if (!firstTap){ window.open(url); }

Save and export your changes as you would normally and it should work after being uploaded to Github.

It should not work while previewing. The link leads to the public website and not the preview website but as long as you're not changing anything drastic, the code should function fine and not require further testing or previewing. Make sure you keep the ' ' around the URL or the code will fail though.


Advanced Customization


Custom Banners

A site-banner must be able to fit both a desktop screen and a mobile phone. Because of this we display banners centered on the middle and with their extra width cut off if the screen isn't wide enough. Because of this, I highly recommend making banners that fit a 16:9 or wider ratio, while keeping the entire title and information within the banner all within the middle.

Making the banner itself in Bootstrap Studio isn't difficult as you can simply copy-paste previous banners (the Kickass banner has the least amount of extra fluff so use that) and change out the image. Changing the image however is a little different from say, image elements as it uses the background-image attribute instead. To edit the image of a banner, open the Appearance tab (the one with the artist palette in the upper right) and scroll to Background. From there click on the pencil next to the current image to select a different image.


Parallax

Parallax Banners require two elements, a version of the banner without parallax and a version with parallax. This might be a little fiddly but you can copy and paste these two elements from the home page (these make up the banner for SY20-21's anthology).

Just make sure to edit the background-image attribute from the Appearance panel. Editing the image on top (by double-clicking the element like you would an image element) won't do anything.

    Reminders

    Only one person can be editing the .bsdesign file of the website at a time. If multiple versions of the file are made with different changes, one of the files will have to be uploaded to the Grafiction GDrive while the other set of changes must either be repeated after the new version is uploaded or abandoned.

    Uploading comics can be done simultaneously by different people, as long as you aren't editing the same comic. The majority of Grafiction's website is within the .bsdesign file used by Bootstrap Studio. The rest of the website is made up of pages generated by the Lue Comic Generator. Since these pages are separate from the beginning they can be safely uploaded and edited by different people as the changes will be merged by Github. Just make sure to assign which comics will be uploaded or edited by whom and to pull from Github whenever a new commit is made.
